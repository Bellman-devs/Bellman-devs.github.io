bellman.environments.transition_model.keras_model.keras.KerasTransitionModel¶

class bellman.environments.transition_model.keras_model.keras.KerasTransitionModel(keras_transition_networks, observation_space_spec, action_space_spec, predict_state_difference=False, observation_transformation=None, trajectory_sampling_strategy=<bellman.environments.transition_model.keras_model.trajectory_sampling.SingleFunction object>, optimizer=<tensorflow.python.keras.optimizer_v2.adam.Adam object>)[source]¶

Bases: bellman.environments.transition_model.transition_model.TransitionModel, Generic[bellman.environments.transition_model.transition_model.T, bellman.environments.transition_model.transition_model.TS]

This class defines transition models which are implemented using Keras. This class acts as an adapter for the TF-Agents configuration objects and data, ensuring that the model definition is separate from the data representation in TF-Agents.

A Keras model consists of one or more sequential feed forward neural networks. The structure of each of the neural networks are defined by each of the KerasTransitionNetwork objects which are passed to the constructor of this class. This class defines the inputs to each of the networks in the ensemble, which have shape:

[batch_dim, features_vector]

where the batch_dim dimension is undefined (set to None), and the features vector by default is the flattened and concatenated latent observations and actions. More precisely, the latent observations and actions tensors are flattened and then concatenated to form the features vector.

The KerasTransitionNetwork objects provide the loss functions, metrics, and other Keras compilation details for the neural networks.

If a custom transition model needs to be trained on a subset of latent observations and/or actions, or certain transformations need to be performed on them, subclasses of this class should implement _get_inputs and _build_model method. Most likely the _step and _train methods will need to be overridden as well.

Parameters

  • keras_transition_networks (List[KerasTransitionNetwork]) – A list of KerasTransitionNetwork objects. The ensemble will consist of this collection of networks.

  • observation_space_spec (BoundedTensorSpec) – The observation spec from the environment.

  • action_space_spec (BoundedTensorSpec) – The action spec from the environment.

  • predict_state_difference (bool) – Boolean to specify whether the transition model should return the next (latent) state or the difference between the current (latent) state and the next (latent) state

  • observation_transformation (Optional[ObservationTransformation]) – To transform observations to latent observations that are used by the transition model, and back. None will internally create an identity transform.

  • trajectory_sampling_strategy (TrajectorySamplingStrategy) – Strategy for propagating the elements of the batch through the ensemble.

  • optimizer (Union[OptimizerV2, str]) – An optimizer for the networks in the ensemble.
