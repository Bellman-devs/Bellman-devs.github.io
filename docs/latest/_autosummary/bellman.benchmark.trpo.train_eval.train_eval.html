bellman.benchmark.trpo.train_eval.train_eval¶

bellman.benchmark.trpo.train_eval.train_eval(root_dir, num_environment_steps, num_eval_episodes, eval_interval, summary_interval, env_name, gym_random_seed, random_seed, num_hidden_layers_agent, num_hidden_nodes_agent, discount_factor, lambda_value, max_kl, backtrack_coefficient, backtrack_iters, cg_iters, reward_normalizer, reward_norm_clipping, log_prob_clipping, value_train_iters, value_optimizer, gradient_clipping, debug, steps_per_policy_update, replay_buffer_capacity, use_tf_function)[source]¶

This function will train and evaluate a TRPO agent.

Parameters

  • root_dir – Root directory where all experiments are stored.

  • num_environment_steps – The number of environment steps to run the experiment for.

  • num_eval_episodes – Number of episodes at each evaluation point.

  • eval_interval – Interval for evaluation points.

  • summary_interval – Interval for summaries.

  • env_name – Name for the environment to load.

  • gym_random_seed – Value to use as seed for the environment.

  • random_seed – A component of the environment model that describes the rewards. At the moment only pre-specified reward models are allowed, i.e. agent assumes reward function is known.

  • num_hidden_layers_agent – A model-free agent parameter, used for constructing neural networks for actor and critic. A number of hidden layers in the neural network.

  • num_hidden_nodes_agent – A model-free agent parameter, used for constructing neural networks for actor and critic. A number of nodes in each hidden layer. Parameter is shared across all layers.

  • discount_factor – discount factor in [0, 1]

  • lambda_value – trace decay used by the GAE critic in [0, 1]

  • max_kl – maximum KL distance between updated and old policy

  • backtrack_coefficient – coefficient used in step size search

  • backtrack_iters – number of iterations to performa in line search

  • cg_iters – number of conjugate gradient iterations to approximate natural gradient

  • reward_normalizer – TensorNormalizer applied to rewards

  • reward_norm_clipping – value to clip rewards

  • log_prob_clipping – clip value for log probs in policy gradient , None for no clipping

  • value_train_iters – number of gradient steps to perform on value estimator for every policy update

  • value_optimizer – optimizer used to train value_function (default: Adam)

  • gradient_clipping – clip born value gradient (None for no clipping)

  • debug – debug flag to check computations for Nans

  • steps_per_policy_update – steps between policy updates

  • replay_buffer_capacity – Capacity of the buffer collecting real samples.

  • use_tf_function – If True, use a tf.function for data collection.
