bellman.environments.utils.virtual_rollouts_buffer_and_driver¶

bellman.environments.utils.virtual_rollouts_buffer_and_driver(environment_model, policy, duration)[source]¶

Generating rollouts from an environment model requires creating a coupled replay buffer and driver. This function returns the replay buffer and driver which are configured correctly.

The replay buffer is a FIFO buffer which has a capacity of 2 × (horizon + 1). This is because the TFDriver does not count boundary transitions towards the total number of steps collected and stored in the replay buffer. The shortest possible trajectory is of length 2 ([StepType.FIRST, StepType.LAST]). The TFDriver counts one step in that trajectory, and so the replay buffer must have enough space to avoid losing the initial time_step.

Parameters

  • environment_model (EnvironmentModel) – virtual environment model. This should not be wrapped in a TFTimeLimit.

  • policy (TFPolicy) – policy to use to generate rollouts.

  • duration (int) – Maximum number of steps in a generated trajectory.

Return type

Tuple[ReplayBuffer, Driver, EnvironmentModel]

Returns

A replay buffer and a driver to use for generating virtual rollouts, and the environment model wrapped in a time limit wrapper.
